# Frontend Asset Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire up a Tailwind v4 + DaisyUI v5 CSS build and vendor HTMX 2 so that `./gradlew bootRun` serves a styled, interactive app.

**Architecture:** The `com.github.node-gradle.node` plugin manages its own Node 22 download inside `~/.gradle/nodejs/` — no system Node required locally or in CI. A `buildCss` Gradle task runs `npm run build` before `processResources`, so all JVM build paths (bootRun, build, test, integrationTest) get the CSS and JS files in the classpath automatically. Generated files are gitignored; `package-lock.json` is committed for reproducible installs.

**Tech Stack:** Tailwind CLI 4.3.0, DaisyUI 5.5.20, htmx.org 2.0.10, htmx-ext-sse 2.2.4, com.github.node-gradle.node 7.1.0, Node.js 22.16.0 (auto-downloaded by the plugin).

## Task 2: Create package.json and CSS entry point

**Files:**
- Create: `app/package.json`
- Create: `app/src/css/app.css`
- Generate: `app/package-lock.json` (via `npm install`)

- [ ] **Step 1: Create `app/package.json`**

```json
{
  "name": "kotlin-saas-template-app",
  "version": "1.0.0",
  "private": true,
  "scripts": {
    "build": "mkdir -p src/main/resources/static/css src/main/resources/static/js && tailwindcss -i src/css/app.css -o src/main/resources/static/css/output.css --minify && cp node_modules/htmx.org/dist/htmx.min.js src/main/resources/static/js/htmx.min.js && cp node_modules/htmx-ext-sse/dist/sse.min.js src/main/resources/static/js/htmx-sse.js",
    "watch": "tailwindcss -i src/css/app.css -o src/main/resources/static/css/output.css --watch"
  },
  "dependencies": {
    "@tailwindcss/cli": "4.3.0",
    "daisyui": "5.5.20",
    "htmx.org": "2.0.10",
    "htmx-ext-sse": "2.2.4"
  }
}
```

- [ ] **Step 2: Create `app/src/css/app.css`**

Create the directory first: `mkdir -p app/src/css`

```css
@import "tailwindcss";
@plugin "daisyui";
@source "../main/resources/templates/**/*.html";
```

The `@source` path is relative to this CSS file. From `app/src/css/app.css`, `../main/resources/templates/` resolves to `app/src/main/resources/templates/`.

- [ ] **Step 3: Generate package-lock.json**

```bash
cd app && npm install && cd ..
```

Expected: `node_modules/` populated, `package-lock.json` created. The `node_modules/` directory is already in `.gitignore`.

- [ ] **Step 4: Commit package.json, package-lock.json, and app.css**


## Task 3: Add node-gradle plugin to Gradle and wire the build

**Files:**
- Modify: `gradle/libs.versions.toml`
- Modify: `app/build.gradle.kts`

- [ ] **Step 1: Add the plugin version and entry to `gradle/libs.versions.toml`**

In the `[versions]` section, add after the last version entry:

```toml
node-gradle              = "7.1.0"
```

In the `[plugins]` section, add after the last plugin entry:

```toml
node-gradle               = { id = "com.github.node-gradle.node", version.ref = "node-gradle" }
```

- [ ] **Step 2: Apply the plugin and add tasks to `app/build.gradle.kts`**

At the top of the file, add the import after the existing imports:

```kotlin
import com.github.gradle.node.npm.task.NpmTask
```

In the `plugins {}` block, add as the last entry:

```kotlin
alias(libs.plugins.node.gradle)
```

After the existing `configurations.all { ... }` block and before the `tasks.register<Test>("unitTest")` block, add:

```kotlin
node {
    version = "22.16.0"
    download = true
}

tasks.register<NpmTask>("buildCss") {
    dependsOn(tasks.npmInstall)
    args.set(listOf("run", "build"))
    inputs.files(
        fileTree("src/css"),
        fileTree("src/main/resources/templates")
    )
    outputs.dir("src/main/resources/static")
}

tasks.processResources {
    dependsOn("buildCss")
}

tasks.clean {
    delete(file("src/main/resources/static"))
}
```

- [ ] **Step 3: Run the tests — expect all three to now pass**

```bash
./gradlew :app:test --tests "tech.sergiodelgado.saastemplate.StaticAssetsTest"
```

Expected: 3 tests PASS. The node-gradle plugin downloads Node 22.16.0 on first run (cached in `~/.gradle/nodejs/` for subsequent runs). Then `npm run build` runs, producing:
- `app/src/main/resources/static/css/output.css`
- `app/src/main/resources/static/js/htmx.min.js`
- `app/src/main/resources/static/js/htmx-sse.js`

If you see a compilation error about `NpmTask` not found: verify the import at the top of `build.gradle.kts` is `import com.github.gradle.node.npm.task.NpmTask`.

- [ ] **Step 4: Commit the Gradle changes**

```bash
git add gradle/libs.versions.toml app/build.gradle.kts
git commit -m "build: add node-gradle plugin and wire Tailwind CSS build into processResources"
```
