# Frontend Asset Pipeline — Design Spec

**Issue:** #53  
**Date:** 2026-06-02  
**Status:** Approved

## Context

Thymeleaf templates reference `/css/output.css`, `/js/htmx.min.js`, and `/js/htmx-sse.js`, but none of these files exist. There is no `static/` directory, no `package.json`, and no build tooling. Every page renders unstyled with broken HTMX interactions.

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| CSS framework | Tailwind v4 + DaisyUI v5 | Current stable; DaisyUI v5 is designed for Tailwind v4 |
| Tailwind build tool | `@tailwindcss/cli` | No PostCSS layer needed; simpler for a Thymeleaf app with no JS bundler |
| HTMX | htmx 2.x + htmx-ext-sse | htmx 2.x is current stable; SSE extension is a separate package |
| Gradle integration | `com.github.node-gradle.node` plugin | Downloads Node automatically — no system Node required locally or in CI |
| Static output | Generated at build time, not committed | Standard practice; keeps the repo clean |

## File Layout

```
app/
├── package.json                             ← new
├── package-lock.json                        ← new, committed
├── src/
│   ├── css/
│   │   └── app.css                          ← new: Tailwind + DaisyUI entry point
│   └── main/resources/
│       ├── static/                          ← new, gitignored
│       │   ├── css/output.css               ← generated by tailwindcss CLI
│       │   └── js/
│       │       ├── htmx.min.js              ← copied from node_modules
│       │       └── htmx-sse.js              ← copied from node_modules
│       └── templates/...                    ← unchanged
```

`app/src/main/resources/static/` is added to `.gitignore`. `package-lock.json` is committed for reproducible installs.

## CSS Entry Point

```css
/* app/src/css/app.css */
@import "tailwindcss";
@plugin "daisyui";
@source "../main/resources/templates/**/*.html";
```

The `@source` directive tells Tailwind where to scan for class names. The path is relative to `app/src/css/app.css`, so it resolves to `app/src/main/resources/templates/`.

## npm Scripts

```json
{
  "scripts": {
    "build": "mkdir -p src/main/resources/static/js && tailwindcss -i src/css/app.css -o src/main/resources/static/css/output.css --minify && cp node_modules/htmx.org/dist/htmx.min.js src/main/resources/static/js/htmx.min.js && cp node_modules/htmx-ext-sse/dist/sse.min.js src/main/resources/static/js/htmx-sse.js",
    "watch": "tailwindcss -i src/css/app.css -o src/main/resources/static/css/output.css --watch"
  }
}
```

`watch` mode is a convenience script for iterative local development (not wired into Gradle).

## Gradle Integration

The `com.github.node-gradle.node` plugin is added to `app/build.gradle.kts`.

```
libs.versions.toml
  node-gradle = "7.1.0"        ← plugin version
  node         = "22.16.0"     ← Node.js LTS version to download
```

npm package versions pinned in `package.json`:

| Package | Version |
|---|---|
| `@tailwindcss/cli` | `4.3.0` |
| `daisyui` | `5.5.20` |
| `htmx.org` | `2.0.10` |
| `htmx-ext-sse` | `2.2.4` |

Task wiring:

```
npmInstall (built-in) ← runs npm ci
    ↓
buildCss (NpmTask)    ← runs npm run build
    ↓
processResources      ← existing Spring Boot task — assembles resources into the JAR
```

`clean` is extended with a `cleanStatic` step that deletes `src/main/resources/static/`.

`bootRun`, `build`, and all test tasks inherit the dependency through `processResources`.

## CI

No changes to `.github/workflows/ci.yml`. The node-gradle plugin caches the Node binary in `~/.gradle/nodejs/`; GitHub Actions' Gradle caching covers it automatically.

## Smoke Test

After implementation, verify manually:

1. `./gradlew :app:bootRun`
2. Open `/dashboard` — DaisyUI `card`, `badge`, `skeleton` components should render
3. Open `/billing` — DaisyUI components should render
4. Dashboard stats grid should poll every 30 s via HTMX
5. Activity feed SSE stream should connect and display events

No automated test is added for the pipeline itself; the existing e2e suite covers end-to-end rendering once the assets exist.
